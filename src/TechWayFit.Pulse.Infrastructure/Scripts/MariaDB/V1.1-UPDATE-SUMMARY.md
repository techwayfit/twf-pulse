# V1.1 Schema Update Summary

## Changes Made

The V1.1 Commercialization Schema script (`00_CommercializationSchema.sql`) has been **updated** to include the V1.2 improvements for flexible plan-based access control.

### What Changed in V1.1 Script

#### Before (Original V1.1)
```sql
CREATE TABLE `ActivityTypeDefinitions` (
    ...
    `RequiresPremium` TINYINT(1) NOT NULL DEFAULT 0,
    `MinPlanCode` VARCHAR(50) NULL,  -- ? Rigid tier hierarchy
    ...
    CONSTRAINT `FK_ActivityTypeDefinitions_MinPlanCode`
        FOREIGN KEY (`MinPlanCode`) REFERENCES `SubscriptionPlans`(`PlanCode`)
)
```

#### After (Updated V1.1 - includes V1.2)
```sql
CREATE TABLE `ActivityTypeDefinitions` (
    ...
    `RequiresPremium` TINYINT(1) NOT NULL DEFAULT 0,
    `ApplicablePlanIds` VARCHAR(500) NULL,  -- ? Flexible plan assignment
    `IsAvailableToAllPlans` TINYINT(1) NOT NULL DEFAULT 0,  -- ? Free activity flag
    ...
    -- No foreign key constraint - more flexible
)
```

### Seed Data Changes

**Before:**
```sql
-- Five Whys: MinPlanCode = 'plan-a' (means "Plan A or higher")
INSERT INTO ActivityTypeDefinitions VALUES
    (..., 1, 'plan-a', ...);  -- ? Enforces hierarchy
```

**After:**
```sql
-- Five Whys: Applicable to Plan A and Plan B specifically
INSERT INTO ActivityTypeDefinitions VALUES
    (..., 1, '00000000-0000-0000-0000-000000000002|00000000-0000-0000-0000-000000000003', 0, ...);
    -- ? Explicit plan list, no hierarchy
```

## Why Update V1.1?

1. **Fresh installations** get the correct schema from the start
2. **No need for V1.2 migration** on new databases
3. **Cleaner architecture** from day one

## Migration Paths

### Path 1: Fresh Installation (New Database)
```bash
# Just run the updated V1.1 script
mysql -u pulseuser -p pulse < V1.1/00_CommercializationSchema.sql
```
? You're done! You have the latest schema.

### Path 2: Upgrade from Old V1.1 (Existing Database)
```bash
# Run the V1.2 migration to update your schema
mysql -u pulseuser -p pulse < V1.2/01_AddApplicablePlansToActivityTypes.sql
```
? Your database is now equivalent to the updated V1.1

## Verification

After applying either path:

```sql
-- Check schema
DESCRIBE ActivityTypeDefinitions;
-- Should show: ApplicablePlanIds, IsAvailableToAllPlans
-- Should NOT show: MinPlanCode

-- Check data
SELECT DisplayName, RequiresPremium, IsAvailableToAllPlans, 
       SUBSTRING(ApplicablePlanIds, 1, 50) AS PlanIds
FROM ActivityTypeDefinitions
WHERE IsActive = 1
ORDER BY SortOrder;
```

**Expected:**
- Free activities: `IsAvailableToAllPlans = 1`, `ApplicablePlanIds = NULL`
- Premium activities: `IsAvailableToAllPlans = 0`, `ApplicablePlanIds = <GUIDs>`
